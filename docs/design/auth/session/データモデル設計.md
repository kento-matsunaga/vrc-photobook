## 1. 扱うデータ

- draft session / manage session の発行履歴
- session の有効期限（`expires_at`）と最終利用時刻（`last_used_at`）
- session の revoke 状態（`revoked_at`）
- token 世代の snapshot（`token_version_at_issue`、manage_url_token 再発行時の一括失効に用いる）
- session_token の hash（**raw token は保存しない**）

raw token そのものや Cookie 値そのものは DB に存在しない。

## 3. `sessions`

`<!-- 付録C P0-14, P0-15, P0-16 -->`

| カラム | 型 | NULL | 既定 | 制約・備考 |
|-------|-----|------|------|----------|
| `id` | `uuid` | NOT NULL | - | PK。**UUIDv7** で生成（ADR-0001） |
| `session_token_hash` | `bytea` | NOT NULL | - | **SHA-256(SessionToken)、32 バイト固定**。raw token は保存しない <!-- 付録C P0-16 --> |
| `session_type` | `text` | NOT NULL | - | CHECK: `'draft' / 'manage'` |
| `photobook_id` | `uuid` | NOT NULL | - | FK → `photobooks.id` ON DELETE CASCADE（Photobook 物理削除時に連鎖削除） |
| `token_version_at_issue` | `int` | NOT NULL | `0` | 発行時の `Photobook.manage_url_token_version` の snapshot。draft session では常に 0 |
| `expires_at` | `timestamptz` | NOT NULL | - | 有効期限。draft: `Photobook.draft_expires_at` まで、manage: 24h〜7d |
| `created_at` | `timestamptz` | NOT NULL | `now()` | 発行時刻 |
| `last_used_at` | `timestamptz` | NULL | - | 最後に有効リクエストで使われた時刻。編集系 API のみ更新 |
| `revoked_at` | `timestamptz` | NULL | - | revoke された時刻。NULL なら有効 |

### 3.1 CHECK 制約

```sql
session_type IN ('draft', 'manage')
expires_at > created_at
last_used_at IS NULL OR (last_used_at >= created_at AND last_used_at <= expires_at)
revoked_at IS NULL OR revoked_at >= created_at
token_version_at_issue >= 0
session_type != 'draft' OR token_version_at_issue = 0   -- I-S5
```

最後の制約（draft session の `token_version_at_issue` は 0 固定）はアプリ層で保証してもよいが、CHECK で守ると事故予防になる。

### 3.2 索引

| 索引 | カラム / 条件 | 用途 |
|------|-------|------|
| PK | `id` | 主キー |
| UNIQUE | `session_token_hash` | I-S4（同一 hash は最大 1 件）。`validate` の高速ルックアップ |
| INDEX | `(photobook_id, session_type, revoked_at)` | `revokeAllDrafts` / `revokeAllManageByTokenVersion` の対象抽出、明示破棄 UI |
| INDEX | `(photobook_id, session_type, token_version_at_issue)` WHERE `revoked_at IS NULL` | reissue 時の旧 version 一括 revoke |
| INDEX | `expires_at` WHERE `revoked_at IS NULL` | 期限切れ session GC |
| INDEX | `revoked_at` WHERE `revoked_at IS NOT NULL` | 30 日経過した revoked session の物理削除 |

### 3.3 重要な設計判断

- **`session_token_hash` は SHA-256（32 バイト固定）**: ADR-0003 の確定事項。raw token は 256bit 以上の暗号論的乱数のため、ストレッチング・ソルト不要 (参照: ADR-0003) <!-- 付録C P0-16 -->
- **`id` は UUIDv7**: ADR-0001。Cookie には載せない。Cookie に載るのは `session_token`（raw 乱数）のみ
- **`session_token` を DB に保存しない**: 漏洩時の被害最小化。検証時は入力 token を SHA-256 ハッシュして照合
- **`photobook_id` への FK + ON DELETE CASCADE**: Photobook 物理削除時に session も削除される（孤児防止）。なお Photobook の論理削除（`status='deleted'`）では session は残るが、Photobook 側で 401 系のレスポンスが返るため実害はない
- **`revoked_at` UPDATE 方式（U1）**: MVP は UPDATE 方式で確定。INSERT-only 履歴方式は Phase 2 で検討
- **`token_version_at_issue` は draft session でも保持**（常に 0）: スキーマ簡素化のため NULL 許容にせず、CHECK 制約で 0 固定にする
- **`last_used_at` は編集系 API でのみ更新**: GET / プレビューでは更新しない。draft の `draft_expires_at` 延長と連動

## 4. テーブル間の責務分担

### 4.1 Photobook 集約との関係

`sessions` は `photobooks.id` を参照するが、**Photobook 集約のテーブル群（photobook_pages, photobook_photos など）には触れない**。

逆方向に、Photobook 集約の UseCase が本テーブルを更新する操作は以下のみ:

| 呼び出し元 | SQL イメージ | 同一 TX |
|----------|-------------|--------|
| Photobook `publishFromDraft` | `UPDATE sessions SET revoked_at=now() WHERE photobook_id=$1 AND session_type='draft' AND revoked_at IS NULL` | ✅ |
| Moderation `reissueManageUrl` | `UPDATE sessions SET revoked_at=now() WHERE photobook_id=$1 AND session_type='manage' AND token_version_at_issue<=$2 AND revoked_at IS NULL` | ✅ |
| Photobook `purge` | `UPDATE sessions SET revoked_at=now() WHERE photobook_id=$1 AND revoked_at IS NULL`（または ON DELETE CASCADE で自動削除） | ✅ |

呼び出し側の TX に参加するため、本テーブル単独の Outbox イベントは発火しない（ドメイン設計 §7）。

### 4.2 upload-verification との関係

`upload_verification_sessions` とは別テーブル。session token の生成・hash 化方針は同じだが、**ライフサイクル・有効期限・利用回数制限が完全に異なる**ため統合しない。

| 観点 | sessions | upload_verification_sessions |
|------|---------|------------------------------|
| 主な役割 | 操作認可 | アップロード許可 |
| 期限 | draft 7d / manage 7d | 30 分 |
| 利用回数制限 | なし | 20 intent |
| photobook_id | 必須 | 必須 |
| token_version_at_issue | あり（manage で使う） | 不要 |

## 5. 保持期間と削除ポリシー

### 5.1 期限切れ session の GC

- `expires_at < now()` の session は無効。検証時に拒否されるが、行は残る
- **自動 reconciler `session_expired_gc`** が `expires_at < now() - INTERVAL '7 days'` の行を物理削除（バッファ 7 日は「期限切れ直後の問い合わせ対応」用）

### 5.2 Revoked session の GC

- `revoked_at IS NOT NULL` の session は無効
- **自動 reconciler `session_revoked_gc`** が `revoked_at < now() - INTERVAL '30 days'` の行を物理削除
- 30 日保持する理由: 監査・紛争時の確認のため

### 5.3 Photobook 削除時

- ON DELETE CASCADE により Photobook 物理削除（purge）時に自動連鎖削除
- 論理削除（`status='deleted'`）では session は残るが、`expireAllForPhotobook(photobookId)` で revoke する選択肢あり（実装判断）

### 5.4 Reconcile（後続）

`docs/design/cross-cutting/reconcile-scripts.md` で詳述する Reconcile スクリプト群:

- 自動 reconciler: `session_expired_gc`、`session_revoked_gc`
- 手動 `scripts/ops/reconcile/`: 必要に応じて `session_audit.sh` 等を後追加

## 6. マイグレーション実施方針

### 6.1 基本

- マイグレーションは 1 機能 1 ファイル
- 破壊的変更は 2 ステップでリリース
- 本テーブルは新規作成のため、初版マイグレーションのみ

### 6.2 初版 DDL（参考）

```sql
CREATE TABLE sessions (
    id                       uuid        PRIMARY KEY,
    session_token_hash       bytea       NOT NULL UNIQUE,
    session_type             text        NOT NULL CHECK (session_type IN ('draft', 'manage')),
    photobook_id             uuid        NOT NULL REFERENCES photobooks(id) ON DELETE CASCADE,
    token_version_at_issue   int         NOT NULL DEFAULT 0 CHECK (token_version_at_issue >= 0),
    expires_at               timestamptz NOT NULL,
    created_at               timestamptz NOT NULL DEFAULT now(),
    last_used_at             timestamptz,
    revoked_at               timestamptz,
    CHECK (expires_at > created_at),
    CHECK (last_used_at IS NULL OR (last_used_at >= created_at AND last_used_at <= expires_at)),
    CHECK (revoked_at IS NULL OR revoked_at >= created_at),
    CHECK (session_type != 'draft' OR token_version_at_issue = 0)
);

CREATE INDEX sessions_revoke_lookup_idx
    ON sessions (photobook_id, session_type, revoked_at);

CREATE INDEX sessions_reissue_lookup_idx
    ON sessions (photobook_id, session_type, token_version_at_issue)
    WHERE revoked_at IS NULL;

CREATE INDEX sessions_expired_gc_idx
    ON sessions (expires_at)
    WHERE revoked_at IS NULL;

CREATE INDEX sessions_revoked_gc_idx
    ON sessions (revoked_at)
    WHERE revoked_at IS NOT NULL;
```

## 7. インフラ層のマッピング指針

[.agents/rules/domain-standard.md](../../../../.agents/rules/domain-standard.md) に従う。ただし本機構は集約ではないため、ディレクトリ名は `auth/session` とする。

```
backend/internal/auth/session/
├── domain/
│   ├── session.go                       # 概念単位 Session
│   └── vo/
│       ├── session_id/                  # UUIDv7
│       ├── session_token/               # raw 256bit 乱数（メモリ中のみ）
│       ├── session_token_hash/          # SHA-256 ハッシュ（DB 保存）
│       ├── session_type/                # draft / manage
│       └── token_version_at_issue/
├── infrastructure/
│   └── repository/rdb/
│       ├── session_repository.go        # 永続化、validate / touch / revoke / revokeAll*
│       ├── marshaller/
│       │   └── session_marshaller.go
│       └── tests/
│           └── session_builder.go
└── internal/
    ├── usecase/
    │   ├── issue_draft_session.go       # Photobook draft 入場から呼ばれる
    │   ├── issue_manage_session.go      # Photobook manage 入場から呼ばれる
    │   ├── validate_session.go          # API Middleware から呼ばれる
    │   ├── touch_session.go             # 編集系 API から呼ばれる
    │   ├── revoke_session.go            # 明示破棄 UI
    │   ├── revoke_all_drafts.go         # Photobook publishFromDraft から呼ばれる
    │   └── revoke_all_manage_by_token_version.go
    └── controller/
        └── auth_session_handler.go      # POST /api/auth/sessions/revoke
```

リポジトリの `revokeAllDrafts(photobookId)` / `revokeAllManageByTokenVersion(photobookId, oldVersion)` は **既存 TX に参加する**仕様（Photobook 集約の TX 内で呼ばれるため）。
