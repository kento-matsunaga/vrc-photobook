# 機能集約ごとのドメイン設計・データモデル設計

本ディレクトリは、[業務知識定義書 v4](../../spec/vrc_photobook_business_knowledge_v4.md)
で定義された業務ルールを、DDD(Domain-Driven Design)の集約単位に分解して設計する。

## 設計原則

1. **業務知識定義書 v4 を唯一の正典とする** — 本設計書は v4 の具現化であり、v4 と矛盾した場合は v4 を正とする
2. **ユビキタス言語の厳守** — v4 第2部の用語を揺らぎなく使用する
3. **集約は小さく保つ** — 集約境界を越える参照はIDのみで行い、結合を避ける
4. **不変条件は集約ルート経由でのみ変更可能にする** — 直接のフィールド変更を禁止
5. **値オブジェクトで型安全性を確保する** — PhotobookId と ManageUrlToken を混同できないようにする
6. **集約間イベントは Outbox で伝搬** — v4 新規の横断原則（§6.11）
7. **整合性は Reconcile で保証** — v4 新規の横断原則（§6.12）

## 非集約の業務要素

| 要素 | 業務知識の対応節 | 扱い |
|------|----------------|-----|
| ランディングページ | §3.9 | 純UI。ドメインなし |
| X共有投稿文生成 | §3.8 | Photobook集約上の読み取り専用ドメインサービス |
| 作成途中データのブラウザ保持 | §6.8 | クライアント責務。v4では server draft が主、ブラウザローカルは補助 |

## 設計書の読み方

各集約ディレクトリは以下の2ファイルで構成される。

- **ドメイン設計.md** — 集約境界、エンティティ、値オブジェクト、不変条件、ドメインサービス、ドメインイベント
- **データモデル設計.md** — テーブル、カラム、制約、インデックス、保持期間ポリシー

各ドキュメント末尾に「v3からの主な変更点」セクションを設けている。
