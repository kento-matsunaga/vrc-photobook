# ディレクトリマッピング

## 概要

コード側ディレクトリと業務知識ドキュメントの対応関係を管理する。
sync-check スキルがこのファイルを参照して乖離を検出する。

## マッピングテーブル

| コード側ディレクトリ | docs側ディレクトリ | 説明 |
|-------------------|-----------------|------|
| `{module}/domain/` | `docs/{業務名}/業務知識.md` | ドメインモデルとビジネスルール |
| `{module}/infrastructure/` | `docs/{業務名}/データモデル.md` | データアクセスとスキーマ |
| `{module}/internal/usecase/` | `docs/{業務名}/ユースケース.md` | 業務フロー |

## 追加方法

1. 新モジュール作成時にこのテーブルに行を追加
2. `docs/{業務名}/` ディレクトリを作成
3. 最低限 `業務知識.md` を作成（後からドメインモデル実装ガイドを追加）
