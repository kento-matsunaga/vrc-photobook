# コーディング基本ルール

## 原則

1. **明示的 > 暗黙的** — マジックナンバー、暗黙の型変換、隠れた副作用を禁止
2. **小さく保つ** — 関数は1つの責務、ファイルは1つの概念
3. **命名で意図を伝える** — コメントが必要なら命名を見直す
4. **エラーは握り潰さない** — エラーは呼び出し元に伝播するか、明示的にハンドリングする

## 禁止事項

- `any` / `interface{}` の安易な使用（型安全性の放棄）
- グローバル変数の変更（テスト不能になる）
- 未使用のimport/変数の放置
- TODO コメントの放置（Issue化するかその場で解決）

## Why

エージェントが「動くコード」を優先し、型安全性やエラーハンドリングを省略する傾向があったため。
特に `interface{}` の多用により、ランタイムパニックが頻発した。
