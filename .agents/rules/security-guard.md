# セキュリティガードルール

## 必須チェック

### 1. 実行者ID取得パターン
操作者のIDはコンテキストから取得する。リクエストパラメータからの取得は禁止。

```go
// ✅ 正しい: コンテキストから取得
executorID := auth.GetUserIDFromContext(ctx)

// ❌ 禁止: リクエストから取得（なりすまし可能）
executorID := req.UserID
```

### 2. テナントスコープガード（RLS）
マルチテナントデータへのアクセスは必ずテナントスコープで制限する。

### 3. 認可チェック統合
ユースケース実行前に権限チェックを必ず行う。

## 禁止事項

- クロステナントデータアクセス
- 認証バイパスのテストヘルパー（テストでも認証フローを通す）
- シークレットのハードコード

## Why

エージェントが以下のセキュリティ問題を引き起こしたため:
- リクエストパラメータから executorID を取得 → 監査ログのなりすましが可能に
- テナントスコープなしのクエリ → 他テナントのデータが漏洩
- テスト用の認証バイパス → 本番コードに混入
