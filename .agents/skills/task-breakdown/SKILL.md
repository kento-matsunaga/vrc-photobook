# タスク分解スキル

## 入力
- 実装対象の仕様（Issue、要件定義、口頭説明）

## 手順

### Step 1: 業務知識の確認
1. `docs/` 配下の関連業務知識ドキュメントを確認
2. 既存の関連コードを読み、現状を把握
3. 不明点があれば質問する（推測で進めない）

### Step 2: タスク分解
以下の粒度に分解する:

```markdown
# タスク: {タスク名}

## Backlog
- [ ] P1: {最優先タスク} — {完了条件}
- [ ] P2: {次優先タスク} — {完了条件}
- [ ] P3: {低優先タスク} — {完了条件}

## In Progress
（実行中のタスクをここに移動）

## Completed
（完了したタスクをここに移動）
```

### Step 3: 依存関係の整理
- タスク間の依存関係を明示
- 並行実行可能なタスクを特定
- ブロッカーがあれば先に解消

## ゴールデンルール

1. **1タスク = 1つの検証可能な成果物** — テストかレビューで検証できること
2. **業務知識先行** — コードを書く前に業務を理解する
3. **TDD原則** — テストを先に書き、実装で通す
4. **漸進的** — 大きな変更を一度にしない
5. **完了条件を明確に** — 各タスクに「何をもって完了か」を書く
6. **スコープクリープ防止** — 追加の発見は別タスクとして切り出す

## 出力
- タスク一覧（Markdownファイル）
- 各タスクの完了条件と優先度
- 依存関係マップ
