# test-failure-capture フック

## 目的

テスト失敗を自動検知し、`harness/failure-log/` に記録して、
failure-to-rule スキルの入力ソースにする。

## フック構成（3点セット）

### 1. afterFileEdit — ファイル変更追跡
- **トリガー**: ソースファイルが編集されたとき
- **動作**: 変更されたファイルを追跡リストに記録
- **目的**: どのファイルが変更されたか把握し、テスト未実行を検知

### 2. afterShellExecution — テスト結果追跡
- **トリガー**: シェルコマンド実行後
- **動作**: テスト実行コマンドの結果を解析
- **検知パターン**:
  - `FAIL` — テスト失敗
  - `PASS` — テスト成功
  - `build failed` — ビルドエラー
- **失敗時**: `harness/failure-log/` に自動記録

### 3. stop — テスト未実行リマインダー
- **トリガー**: エージェント停止時
- **動作**: 変更されたファイルに対応するテストが実行されたか確認
- **未実行時**: 警告メッセージを出力

## 設定方法

`.claude/settings.json` の `hooks` セクションに設定を追加する。
詳細は `.claude/settings.json` を参照。

## セキュリティ考慮事項

- フックスクリプトはリポジトリ内に配置（バージョン管理対象）
- 外部コマンドの実行は最小限に
- 機密情報をログに出力しない
