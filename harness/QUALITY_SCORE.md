# 品質スコア

## 概要

モジュール × レイヤーごとの品質を可視化し、改善優先度を判断する。

## スコア基準

| グレード | 基準 |
|---------|------|
| A | ルール準拠、テスト充実、ドキュメント整合 |
| B | 概ね準拠、軽微な改善点あり |
| C | 一部ルール違反あり、テスト不足 |
| D | 重大なルール違反、テスト欠如 |
| F | 未評価 / 評価不能 |

## スコアボード

| モジュール | ドメイン | インフラ | ユースケース | コントローラー | テスト | ドキュメント | 総合 |
|-----------|---------|---------|------------|-------------|-------|------------|------|
| (例) auth | B | C | B | B | C | D | C |

## 改善履歴

| 日付 | モジュール | レイヤー | Before → After | 理由 |
|------|-----------|---------|---------------|------|
| | | | | |

## 使い方

1. 新モジュール追加時にスコアボードに行を追加
2. self-verification 後にスコアを更新
3. C以下のセルを優先的に改善
4. 改善時は履歴に記録
